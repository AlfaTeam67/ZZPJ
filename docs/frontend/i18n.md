# 🌐 Internationalization (i18n)

## Przegląd

Fin-Insight obsługuje dwa języki: **polski (PL)** i **angielski (EN)**. Implementacja oparta na `react-i18next`.

## Stack

- `i18next` — silnik tłumaczeń
- `react-i18next` — integracja z React (hook `useTranslation`)
- `i18next-browser-languagedetector` — automatyczna detekcja języka przeglądarki

## Struktura plików

```
src/i18n/
├── config.ts              # Inicjalizacja i18next
├── types.ts               # Module augmentation (type-safe keys)
├── hooks/
│   └── useLanguage.ts     # Hook: language, setLanguage, locale
└── locales/
    ├── pl/
    │   ├── common.json    # Wspólne (przyciski, etykiety)
    │   ├── nav.json       # Nawigacja
    │   ├── auth.json      # Logowanie
    │   ├── portfolio.json # Ekran portfela
    │   ├── dashboard.json # Dashboard
    │   ├── market.json    # Dane rynkowe
    │   └── advisor.json   # Doradca AI
    └── en/
        └── ... (identyczna struktura)
```

## Jak dodać nowe tłumaczenie

1. Dodaj klucz do odpowiedniego pliku JSON w `locales/pl/` i `locales/en/`
2. Użyj w komponencie:

```tsx
import { useTranslation } from 'react-i18next'

function MyComponent() {
  const { t } = useTranslation('portfolio') // namespace
  return <h1>{t('title')}</h1>
}
```

## Konwencje kluczy

- **Namespace**: kebab-case (`portfolio`, `market`, `common`)
- **Klucze**: kebab-case z dot-notation dla zagnieżdżeń (`create-title`, `col-date`)
- **Interpolacja**: `{{variable}}` — np. `"units": "{{count}} szt."`
- **Pluralizacja**: sufiks `_one`, `_few`, `_many` (PL) lub `_one`, `_other` (EN)

## Detekcja języka

Kolejność: `localStorage` → `navigator.language` → fallback `pl`

Klucz localStorage: `fin-insight.lang`

## Hook `useLanguage`

```tsx
import { useLanguage } from '@/i18n/hooks/useLanguage'

const { language, setLanguage, locale } = useLanguage()
// language: 'pl' | 'en'
// locale: 'pl-PL' | 'en-US' (do formatowania liczb)
```

## LanguageSwitcher

Komponent `<LanguageSwitcher />` renderuje toggle PL/EN. Umieszczony w:
- `AppHeader` (główna nawigacja)
- `LoginPage` (nagłówek)

## Format liczb

`formatCurrency`, `formatPercent`, `formatSignedCurrency` z `utils/formatNumber.ts` przyjmują opcjonalny parametr `locale` (`'pl-PL'` | `'en-US'`). Domyślnie `'pl-PL'`.

Komponenty przekazują locale z `useLanguage().locale` — utils nie importuje i18next.
